## 2. 已完成内容

### 2.1 架构与基础设施

- ASP.NET Core Minimal API + EF Core Code First + SQLite
- Swagger / OpenAPI + 前端 openapi-generator 客户端
- Docker 单机部署配置
- SQLite 备份与恢复脚本
- 17 次数据库 Migration，模型逐步演化

### 2.2 账号与用户上下文

- 注册 / 登录 / 登出 / 全部退出 / 多会话管理
- 会话撤销、过期清理
- 密码强度校验、邮箱格式校验
- 前端 Token 持久化与登录态恢复

### 2.3 团队、成员与权限

- 团队 CRUD、切换、团队级访问控制
- 真人成员邀请（创建/接受/撤销/过期）
- AI 成员创建 + 岗位模板快速绑定
- 成员角色修改、移除、角色说明

### 2.4 业务主链

- 项目 CRUD + 阶段/摘要/风险/下一步 + 参与成员关系 + 聚合视图
- 坐台程序 CRUD + 欢迎语/FAQ/建议提问/服务时间/渠道/必填校验
- 客户 CRUD + 标签/跟进状态/最近联系时间/关联项目/来源
- 会话 CRUD + 详情/消息/自动建单 + 自动触发协作
- 工单 CRUD + 分类/优先级/负责人/SLA/活动时间线/评论/解决结果
- 公开坐台程序客户入口 + 自助提交

### 2.5 AI 多智能体协作

- 工单触发 / 会话触发 / 项目触发三种协作场景
- 结构化步骤输出（含 Schema 版本跟踪）
- AI 实时 LLM 生成步骤摘要与整体摘要
- **结果自动回写**：工单活动记录、会话 AI 摘要消息、客户跟进状态、项目摘要/风险/下一步
- 工作流模板库（客户接待链/工单分诊链/项目跟进链）
- 手动触发与自动触发区分

### 2.6 AI 岗位模板

- 内置模板库（前台接待/工单协调/项目助理）可通过 `/api/ai-member-templates` 查询
- 团队级模板 CRUD（创建/编辑/启用/禁用/克隆）
- 模板字段：职责摘要、权限边界、系统提示词、允许工具集、可执行动作、知识域、自主性

### 2.7 外部系统集成

- 统一 Provider 接口层（IFileKnowledgeProvider / ICustomerProvider / IProjectProvider / ITicketProvider / ITaskProvider）
- Nextcloud 适配器（WebDAV 文件列表 + 元数据读取 + 连接校验）
- ERPNext 适配器（REST API 认证 + 客户/项目/工单/任务预览 + 导入 + 冲突处理）
- 团队级连接配置、健康校验、同步审计日志
- 前端集成面板：连接管理、预览、导入、映射视图、同步状态

### 2.8 前端工作台

- 左侧管理区（团队设置 + 成员管理 + 集成管理）
- 左侧业务区（项目 / 坐台程序 / 客户 / 会话 / 工单）
- 右侧聊天区
- 公开坐台页面
- OpenAPI 生成的类型安全客户端
- 中文格式化工具（所有枚举状态展示）

### 2.9 工程与运维

## 4. 当前主要缺口

### 4.1 P3-4 可交付化（未启动）

- 页面结构升级：从单页工作台升级为多页路由
- 可用性优化：空状态 / 加载态 / 错误提示 / 搜索筛选 / 统一状态样式
- 交付准备：演示账号与演示数据 / 试用说明 / 部署流程固化

### 4.2 工程与质量

- `Program.cs` 5133 行，68 个端点全在一文件，维护成本高
- 测试覆盖不足（68 端点仅 22 行测试文件）
- 前端缺少 Loading 统一管理、表单校验库
- 无 CI/CD 流水线
- 无 API 版本化策略
- 109 个文件变更未提交 / 未分阶段

### 4.3 产品深度

- 坐台程序编辑界面仍偏基础
- 工单处理动作（分配/状态变更/解决提交）的 UI 流程可优化
- 聊天尚未与业务对象深度联动（上下文中直接创建/跳转）
- AI 协作结果的 Prompt 质量需要真实数据迭代

### 4.4 集成深度

- Nextcloud 文件内容索引 / 全文检索未接
- ERPNext 写入粒度有限（仅导入，未做双向同步）
- 连接凭据管理使用明文存储，正式环境需要加密方案

## 5. 建议优先级

### P3-4 可交付化

建议作为当前唯一优先级。

#### P3-4-1 页面结构升级

从"单页工作台"升级为更合理的信息架构。建议使用前端路由区分：
- `/login` — 登录 / 注册
- `/workspace` — 主工作台
- `/concierge/:id` — 公开坐台程序入口
- `/admin/team` — 团队管理
- `/admin/integrations` — 集成管理
- `/business/projects` — 项目页
- `/business/customers` — 客户页
- `/business/tickets` — 工单页

#### P3-4-2 可用性优化

- 完善空状态、加载态、错误态的 UI 展示
- 增加列表搜索和筛选
- 更清晰的操作反馈与成功/错误提示
- 统一状态标签、按钮、表单样式

#### P3-4-3 交付准备

- 演示账号与种子数据脚本
- 试用说明文档
- 环境隔离规范固化
- CI/CD 基础配置

### P4 工程加固（P3-4 之后）

建议在 P3-4 完成后启动。

#### P4-1 测试基础建设

- 补齐核心 API 集成测试（当前仅覆盖注册/登录/团队/工单工作流）
- 服务层单元测试（AgentWorkflowOrchestrator / ExecutionService / WritebackService）
- 前端组件测试（可选）

#### P4-2 代码结构优化

- 拆分 `Program.cs`：按领域拆为独立文件或 Controller
- 统一前端 Loading / 错误处理抽象
- 引入表单校验库

#### P4-3 生产化

- 正式认证策略（JWT 刷新、RBAC 细化）
- 连接凭据加密存储
- API 限流与安全头
- 监控与告警基础
- CI/CD pipeline

## 6. 现状总结

项目从"聊天 Demo"起步，到目前已完整实现：

- 一套可操作的业务主链（团队 → 项目 → 坐台 → 客户 → 会话 → 工单）
- 一套 AI 多智能体协作引擎（触发 → 编排 → LLM 生成 → 结果回写）
- 一套外部系统集成框架（Nextcloud / ERPNext 适配器模式）
- 一个可操作的前端工作台

当前最重要的事不再是添加新能力，而是：

> 把原型工作台打磨成真正可试用的产品版本，同时开始为代码结构和测试建立可持续工程基础。
