## 3. P3-1 真实业务化（✅ 已完成）

### 3.1 坐台程序业务配置

已实现：
- 欢迎语配置、FAQ 范围、服务时间、渠道标识
- 填写指引、建议提问、邮箱/手机号必填规则
- 自动建单策略、转人工策略字段
- 前端创建表单与资源展示
- 公开客户入口与自助提交（含手机号/邮箱必填校验）

### 3.2 客户资产增强

已实现：
- 客户标签、跟进状态、最近跟进时间、关联项目
- 客户来源渠道、外部系统映射
- 客户详情编辑区 + 后端更新接口
- 工单数量聚合展示

### 3.3 工单处理增强

已实现：
- 工单分类、期望处理时间、最近活动时间
- 工单活动记录模型（TicketActivity）
- 活动时间线 + 评论入口
- 状态变更 / 负责人变更 / 优先级变更记录
- 解决结果 + 完成时间
- SLA 视图（进行中 / 已超时 / 已按时完成 / 已超时完成）

### 3.4 项目协同增强

已实现：
- 项目阶段、摘要、风险、下一步（字段 + Migration）
- 项目参与成员关系（负责人 + 参与成员维护）
- 项目聚合视图（客户数 / 工单数 / 参与成员）
- 项目详情编辑入口

## 4. P3-2 真实集成化（✅ 已完成）

### 4.1 Nextcloud 真实接入

已实现：
- WebDAV `PROPFIND` 连接校验 + 文件列表读取 + 文件元数据预览
- 指定知识目录并刷新预览
- 前端工作台展示文件预览结果

### 4.2 ERPNext 真实接入

已实现：
- `frappe.auth.get_logged_user` 认证校验
- 客户 / 项目 / 工单 / 任务真实 REST API 读取与预览
- 第一条写入闭环：预览客户 → 导入为本地客户资产
- 第二条写入闭环：预览项目 → 导入为本地项目资产
- 第三条写入闭环：预览工单 → 导入到当前选中本地项目
- 基础冲突处理：首次导入 vs 已导入后的同步更新已区分

### 4.3 同步与映射策略

已实现：
- 外部 ID 与本地 ID 映射总览
- 首次导入 vs 强制同步更新区分
- 重复导入冲突 → 写入集成审计日志
- 同步健康摘要、最近冲突、最近同步记录展示
- 外部预览项和本地映射对象之间直接跳转

## 5. P3-3 真实 AI 化（✅ 已完成）

### 5.1 岗位模板升级

已实现：
- 内置标准岗位模板库（前台接待 / 工单协调 / 项目助理）
- 后端 `/api/ai-member-templates` 模板库接口（支持系统级 + 团队级）
- 团队级模板 CRUD（创建/编辑/启用/禁用/克隆）
- 模板字段：职责摘要、权限边界、系统提示词、允许工具集、可执行动作、知识域、自主性
- 前端快速创建可直接选择模板

### 5.2 任务触发机制

已实现：
- 工单触发多智能体协作链
- 会话触发（项目触发接口，支持不同业务对象）
- 协作记录列表按工单 / 会话 / 项目上下文切换
- 前端在工作项详情页直接发起协作
- 触发方式区分（手动触发 `Manual` / 自动触发 `Auto`）
- 会话创建后自动触发协作
- 工单创建后自动触发协作

### 5.3 结果回写

已实现：
- 工单协作结果 → 写入工单活动记录 + 更新最近活动时间
- 会话协作结果 → 生成 AI 协作摘要消息 + 更新客户最近联系时间与备注
- 项目协作结果 → 更新项目摘要、风险摘要与下一步建议
- 前端在触发协作后自动刷新相关详情

### 5.4 工作流模板

已实现：
- 工作流模板库接口，按作用域返回项目/会话/工单三类模板
- 前端统一协作卡片，支持快速套用模板
- 三类基础模板：客户接待链、工单分诊链、项目跟进链
- 结构化步骤输出（Schema 版本 `workflow_step_v1` / `workflow_summary_v1`）
- AI LLM 实时生成步骤摘要和整体摘要
- 输出原始响应与尝试追踪

## 6. P3-4 可交付化（🔲 未启动）

### 6.1 目标

把当前原型工作台升级为可试用的产品。

### 6.2 建议拆分

#### P3-4-1 页面结构升级

建议将当前单页工作台升级为路由驱动的多页结构：

- `/login` — 登录 / 注册
- `/workspace` — 主工作台
- `/concierge/:id` — 公开坐台程序入口
- 按领域拆分管理区与业务区为独立页面

#### P3-4-2 可用性优化

- 完善空状态 / 加载态 / 错误态的 UI 展示
- 增加列表搜索与筛选
- 统一的操作反馈
- 统一状态标签与样式
- 导航结构优化

#### P3-4-3 交付准备

- 演示账号与种子数据
- 试用说明文档
- 部署流程固化
- CI/CD 基础配置

### 6.3 建议执行顺序

1. 页面结构升级（路由 + 布局拆分）
2. 可用性优化（空状态 + 样式统一）
3. 交付准备（种子数据 + 文档）
